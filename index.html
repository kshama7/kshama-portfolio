Portfolio Platform

Prod-grade DRL + classical optimization · live · source

Five deep-RL agents (PPO, A2C, DDPG, SAC, TD3) and four classical optimizers (Markowitz, Min-Variance, Max-Sharpe, HRP) that allocate US-stock portfolios, served behind a FastAPI inference API and a Next.js dashboard. Distributed systems · fault tolerance

distsched — Distributed Job Scheduler

Go · gRPC · leader election · failure recovery · source

A fault-tolerant distributed job scheduler in Go. A cluster of schedulers elects a leader (Raft-style majority vote with term fencing), and the leader dispatches tasks to a pool of workers that run them as subprocesses — with a priority queue, exponential-backoff retry, a dead-letter queue, DAG dependencies, and delayed/cron schedules.

Built for resilience: metadata replicates to followers so a new leader takes over on failover, a reaper requeues the work of dead workers and expired leases, and orphaned jobs are reclaimed on restart. A Docker Compose cluster and a chaos script prove failover by killing the leader under load — and the docs are deliberately honest about what's implemented versus cut.
